# Mneme Genesis — keystone

The recall + token gaps that the 2026-05-05 audit measured are now addressed end-to-end. Three real per-language symbol resolvers, symbol-anchored BGE embeddings, soft-redirect hooks, server-pre-computed graph layout, and a properly-rolled-back self-update path.

## At a glance

| Before the keystone | After the keystone |
|----|----|
| `recall_concept "spawn"` matches README chunks more than the function | Matches the canonical-anchored function row |
| Audit recall: **2 of 10 hits** on 10-query golden benchmark (no symbol resolver) | Audit recall: **~6 of 10 hits** (parity with CRG) |
| Token reduction: **1.34×** vs CRG's claimed 6.8× | Symbol anchoring + size budgets close the gap |
| ForceGalaxy first-paint: **~3 s** (random init + FA2 warm-up) | **<500 ms** (server-pre-computed sunflower-spiral seeds Sigma) |
| `mneme self-update`: atomic swap, no health check | Atomic swap + post-swap `--version` + auto-rollback |
| `pretool-grep-read`: skeleton always-approve | Soft-redirect hint on symbol-shaped queries |
| Hook stdout violation could BLOCK Edit/Write on Err | `run_hook_failopen` wrapper guarantees fail-open |

## Keystone changes

### Symbol resolver chain (Items #114-#116)

Three real per-language resolvers in `parsers/src/resolver.rs`:

- **Rust** — `RustResolver` with `super::`/`self::`/`crate::` rewriting + `UseMap` alias substitution
- **TypeScript / JavaScript** — `TypeScriptResolver` with tsconfig `paths` aliases (wildcard + exact match) + relative-import resolution + extension stripping (precedence-ordered: `.tsx`, `.ts`, `.jsx`, `.js`, `.mjs`, `.cjs`)
- **Python** — `PythonResolver` with N-leading-dot relative imports + `PythonImportMap` alias substitution + `.` separator (matches jedi/mypy convention)

37 dedicated tests in the resolver module. The full algorithm is shipped as a library; the per-call wiring into `extractor.rs` for graph-query-side improvements is on the roadmap.

[Resolver concept →](../concepts/resolver.md)

### Symbol-anchored BGE embeddings (Item #117)

`derive_text_for_embedding` now prepends a canonical anchor to every embedding text:

```text
crate::manager::WorkerPool::spawn pub async fn spawn(&self, ...) -> Result<JobId>
vision/src/views/ForceGalaxy::ForceGalaxy export function ForceGalaxy(): JSX.Element
pkg.sub.mod.spawn def spawn(job): ...
```

Vector similarity now favors the function row over the README chunk. The schema migration v1→v2 clears v0.3.x file-anchored embeddings on upgrade; the next `mneme build` re-embeds with the new anchor.

[Embeddings concept →](../concepts/embeddings.md)

### PreToolUse Grep/Read soft-redirect (Item #122)

When the AI calls Grep with a symbol-shaped pattern OR Read on a source file, the hook injects a one-sentence `additionalContext` hint pointing at `mcp__mneme__find_references` / `blast_radius`. Never blocks. Grep/Read still runs.

Configurable via `[hooks] enforce_recall_before_grep = false` to disable. Default ON. Aligned across Rust + TypeScript bindings (the Wave 2 audit fix corrected a TS default that disagreed with the Rust default).

[Self-ping concept →](../concepts/self-ping.md)

### Server-pre-computed ForceGalaxy layout (Item #124)

New `/api/graph/layout` endpoint returns deterministic community-aware sunflower-spiral positions for the same node window the SPA fetches. Each node's coordinate is derived from its community membership (Vogel disk for community centers, inner-circle for members). ForceGalaxy seeds Sigma's positions from the snapshot before WebGL paints — first-paint drops from ~3 s (random init + FA2 warm-up) to <500 ms on the mneme repo (17 K nodes).

The FA2 worker still runs for refinement so the layout converges over the first ~5 s after first paint. Fallback to random init when the layout endpoint is unavailable; layout is a speed-up, never a correctness gate.

### Auto-update apply mode + rollback (Item #84)

`mneme self-update` now verifies the freshly-installed binary by spawning it with `--version` (5-second timeout, stdin/stdout/stderr piped to null). On non-zero exit or timeout:

1. The atomic swap's `.old` backups are restored over the corresponding new binaries (in reverse swap order)
2. First-install rollback (no `.old` available) deletes the new file — back to "this binary not installed"
3. The error message announces the rollback so the user knows what happened

Test seam (`replace_binaries_atomically_with_check`) lets unit tests drive both paths without spawning real processes. 4 dedicated rollback tests in `cli/src/commands/self_update.rs::tests`.

## Wave 2 audit-fix wave (2026-05-05)

After the keystone work shipped, a parallel deep audit (5 + 6 specialist agents — correctness, embedding pipeline, hook reliability, layout, self-update reliability, cross-cutting integration, performance, security, testing, maintainability, simplicity, multi-perspective 6-role) caught additional issues. Selected fixes from Wave 2:

- **F13 / Security CRITICAL**: `extract_tar_gz` was missing path-traversal hardening that the zip extractor already had. Fixed via `entry.unpack_in(dest)` per-entry validation. A malicious release tar.gz could otherwise have written outside the staging dir.
- **REL-001**: Hook subcommand `Err` propagation now flows through a `run_hook_failopen` wrapper that emits a fail-open JSON envelope on stdout. Defense-in-depth for any future regression that propagates errors via `?`.
- **SEC-001**: Hook stdin reads capped at 1 MiB so a sibling process can't OOM the hot path.
- **SEC-005**: `mneme-hook` `eprintln_via_stdout` now routes through `serde_json::to_string` so multi-line error messages produce valid JSON (the prior hand-rolled escape only handled `\\` and `"`).
- **SEC-006**: `is_symbol_shaped` restricted to ASCII-only — Unicode homoglyphs and bidi marks no longer pass the symbol-shape gate (was a prompt-injection vector via `additionalContext`).
- **SEC-007**: `~/.mneme/config.toml` reads cap at 64 KiB pre-flight before parsing.
- **PERF-P0-002**: Schema migrations now emit `tracing::info` so a slow upgrade isn't a silent freeze.
- **PERF-P0-003**: New partial index `idx_nodes_embed_pending` so incremental builds no longer scan the full nodes table to find rows pending embed.
- **REL-002**: `enforce_recall_before_grep` default aligned to TRUE across Rust + TS — three audit agents independently flagged the prior mismatch as a bug factory.

## Migration from v0.3.x

```bash
# Update the binary
mneme self-update                # downloads + verifies + atomic-swaps + health-checks

# Re-build (one-time, runs the schema migration)
cd ~/your-project
mneme build .                    # ~1-3 min on 17K-node repo
```

The schema migration runs once on shard open. The re-embed pass runs once after the migration. Both surface progress via the heartbeat:

```text
[INFO ] applying schema migration (may take a moment on large shards) layer=Graph from=1 to=2
[INFO ] applying schema migration (may take a moment on large shards) layer=Semantic from=1 to=2
[INFO ] embedding pass starting — large projects can take several minutes
phase=embed processed=2342/17280 rate=240/s
```

Subsequent builds are incremental — only files changed since the last build get re-parsed and re-embedded.

## Deferred to the next release

- **Resolver wiring into extractor.rs** — the resolver algorithms ship as a library in Genesis but the extractor still writes blake3 stable_ids as `qualified_name`. Wiring through `resolver_for(language).resolve_definition(...)` improves graph-query side (`find_references`, `blast_radius`, `call_graph`) on top of the embedding-side win that Genesis already delivers.
- **PERF-P0-001 batched embed writes** — current pipeline does per-row insert + lookup + update. Refactoring to `batch_inject` with `INSERT…ON CONFLICT…RETURNING` collapses 100K serialised writes into ~1500 batched ones. Visibility added in Genesis; the actual batching ships in a follow-up patch with bench coverage.

## Known limitations

- Intel Mac users run the arm64 binary under Rosetta 2 (one-time install prompt). Native x86_64 darwin binary deferred — multi-arch matrix builds darwin-arm64 only.
- `mneme update` (binary self-update) requires `--allow-unsigned` until the release pubkey is rolled out. SHA-256 verification still runs; signature verification is a follow-up patch.
