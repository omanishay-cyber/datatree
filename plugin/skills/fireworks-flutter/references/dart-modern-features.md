# Dart Modern Features Reference (3.7 - 3.10)

> Language features from Dart 3.7 through 3.10, covering wildcards, null-aware collections,
> dot shorthands, augmentations, build hooks, and the new formatter.

---

## 1. Dart 3.7 — Wildcards and New Formatter

### Wildcard Variables (Binding `_`)

In Dart 3.7+, any local variable or parameter named `_` is a wildcard — it does not bind a value and cannot be referenced. Multiple `_` declarations in the same scope are allowed.

```dart
// Before 3.7: second `_` was a redeclaration error
// After 3.7: both are wildcards, no conflict
void processRecords(List<(String, int, bool)> records) {
  for (final (name, _, _) in records) {
    print(name); // Only `name` is bound
  }
}

// Wildcard parameters — signal "not used"
Future<void> fetchData(
  String url, {
  void Function(int _, int _)? onProgress,
}) async {}

// Wildcard catch variables
try {
  await riskyOperation();
} catch (_) {
  handleGenericError(); // Exception object deliberately unused
}

// Wildcard in switch patterns
final result = switch (shape) {
  Circle(radius: var r, _) => 'Circle r=$r',
  Square(side: var s)      => 'Square s=$s',
  _                        => 'Unknown shape',
};
```

### New Dart Formatter (dart format)

Dart 3.7 shipped a completely rewritten formatter with improved line-breaking heuristics.

```dart
// Trailing commas force multi-line formatting:
final widget = Container(
  padding: const EdgeInsets.all(16),
  decoration: BoxDecoration(
    color: Colors.blue,
    borderRadius: BorderRadius.circular(12),
  ), // trailing comma forces multi-line
);

// Without trailing comma, formatter may keep it single-line if it fits:
final widget = Container(padding: const EdgeInsets.all(16));
```

```bash
dart format lib/ test/
# Check without modifying:
dart format --output=none --set-exit-if-changed lib/
```

## 2. Dart 3.8 — Null-Aware Collections and Cross-Compilation

### Null-Aware Elements in Collections

Spread and add elements conditionally based on nullability, without explicit `if` checks.

```dart
// ?element — include only if non-null
String? subtitle;
String? badge;

final widgets = <Widget>[
  Text(title),
  ?Text(subtitle),   // included only when subtitle != null
];

// ?spread — spread only if non-null
List<String>? extraTags;
final allTags = <String>[
  'flutter',
  'dart',
  ...?extraTags,
];

// Null-aware in maps
String? optionalKey;
final headers = <String, String>{
  'Content-Type': 'application/json',
  ?optionalKey: 'value',  // entry included only if key is non-null
};

// Null-aware in sets
int? optionalId;
final ids = <int>{1, 2, 3, ?optionalId};
```

### Cross-Compilation Support

```bash
dart compile exe bin/tool.dart --target-os=linux
dart compile exe bin/tool.dart --target-os=macos
dart compile exe bin/tool.dart --target-os=windows
```

## 3. Dart 3.10 — Dot Shorthands, Augmentations, and Build Hooks

### Dot Shorthands (Static Access Shorthand)

When the expected type is known from context, omit the type name and use a leading dot.

```dart
// Before: verbose
final color = Color.fromARGB(255, 66, 133, 244);
final alignment = Alignment.center;
final mode = ThemeMode.dark;

// After: dot shorthand when type is known
Color color = .fromARGB(255, 66, 133, 244);
Alignment alignment = .center;
ThemeMode mode = .dark;

// In function arguments where parameter type is known
void setTheme(ThemeMode mode) {}
setTheme(.dark);

// In switch expressions
String describe(ThemeMode mode) => switch (mode) {
  .system => 'System default',
  .light  => 'Light theme',
  .dark   => 'Dark theme',
};

// In widget trees — massive verbosity reduction
Widget build(BuildContext context) {
  return Container(
    padding: .all(16),                // EdgeInsets.all(16)
    margin: .symmetric(horizontal: 8), // EdgeInsets.symmetric(...)
    decoration: BoxDecoration(
      color: .blue,                    // Colors.blue
      borderRadius: .circular(12),     // BorderRadius.circular(12)
    ),
  );
}

// Collection literals
List<Color> palette = [.red, .green, .blue, .fromARGB(255, 128, 0, 255)];
```

**Limitations:**
- Only works when the type is statically known (declared variable type, parameter type, return type)
- Does not work with `var` or `final` without a type annotation
- Does not work with `dynamic`

### Augmentations (Replacement for Cancelled Macros)

Augmentations allow a library to add members to classes defined in another file, as long as both are part of the same library. This replaces the cancelled macros feature.

```dart
// --- user.dart ---
import 'package:json_annotation/json_annotation.dart';
part 'user.g.dart'; // augmentation file generated by build_runner

@JsonSerializable()
class User {
  final String id;
  final String name;
  final String email;

  User({required this.id, required this.name, required this.email});

  factory User.fromJson(Map<String, dynamic> json) => _$UserFromJson(json);
  Map<String, dynamic> toJson() => _$UserToJson(this);
}
```

**Why macros were cancelled:**
- Unpredictable compilation ordering
- IDE support was extremely challenging
- Non-deterministic build times
- Augmentations provide same benefits with predictable, cacheable builds

**Impact on existing codegen:**
- `json_serializable`, `freezed`, `riverpod_generator` all continue working
- They will migrate to augmentation syntax internally over time
- Your `part 'file.g.dart'` pattern stays the same

### Build Hooks (dart pub hooks)

Build hooks let packages run Dart scripts during build time for native code compilation.

```yaml
# pubspec.yaml
hooks:
  build: hook/build.dart
```

```dart
// hook/build.dart
import 'package:native_assets_cli/native_assets_cli.dart';

void main(List<String> args) async {
  await build(args, (config, output) async {
    final sources = config.packageRoot.resolve('src/native.c');
    // compile and link native code
  });
}
```

## 5. Migration Tips

### Adopting Dot Shorthands Incrementally

```dart
// Start with enums — highest readability win
// Before: ThemeMode.dark, Brightness.light, MainAxisAlignment.center
// After:  .dark, .light, .center

// Then constructors in widget trees
// Before: EdgeInsets.all(16), BorderRadius.circular(8)
// After:  .all(16), .circular(8)

// Avoid in ambiguous contexts
var x = .something; // ERROR: no type context
```

### Adopting Null-Aware Collections

```dart
// Before
final items = <Widget>[
  const Header(),
  if (subtitle != null) Text(subtitle!),
  if (badge != null) Badge(label: badge!),
  const Footer(),
];

// After
final items = <Widget>[
  const Header(),
  ?Text(subtitle),
  ?Badge(label: badge),
  const Footer(),
];
```
